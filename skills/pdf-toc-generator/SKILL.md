description: 
  为一个没有目录的PDF文件自动生成目录（Table of Contents），通过调用fitz（PyMuPDF）库解析PDF中的标题结构，提取章节信息并输出目录。

## 执行流程

### 1. 确认用户需求与输入

- 获取用户提供的PDF文件路径（本地路径或URL）
- 确认用户是否需要自定义目录层级深度（默认自动检测）
- 确认用户是否需要输出格式选项（纯文本目录 / Markdown格式 / JSON格式）

### 2. 检查环境依赖

确保以下Python库已安装：

```bash
pip install pymupdf  # fitz库
```

如果未安装，先执行安装命令。

### 3. 加载PDF并分析文档结构

使用 fitz 库打开PDF文件，遍历每一页，提取文本块信息。

**核心分析逻辑：**

#### 3.1 提取文本块

对每一页，使用 `page.get_text("dict")` 获取页面上的文本块（blocks），每个block包含：
- `type`: 文本类型（0=文本, 1=图片）
- `bbox`: 边界框坐标 (x0, y0, x1, y1)
- `lines`: 文本行列表，每行包含：
  - `spans`: 文本片段列表，每个span包含：
    - `text`: 文本内容
    - `size`: 字体大小
    - `flags`: 字体样式标志（粗体、斜体等）
    - `font`: 字体名称
    - `color`: 字体颜色

#### 3.2 识别标题

通过以下特征识别标题：

1. **字体大小特征**：标题通常比正文大
   - 统计全文字体大小分布，找出正文基准字体大小
   - 大于正文基准字体大小 1.2 倍以上的文本块视为潜在标题
   
2. **字体样式特征**：
   - 粗体（flags & 2^4）
   - 斜体（flags & 2^6）
   - 特定标题字体（如 "Bold"、"Heavy" 等字样的字体名）

3. **位置特征**：
   - 标题通常在页面顶部区域（y坐标较小）
   - 标题通常居中或左对齐

4. **内容特征**：
   - 标题通常较短（一般不超过100个字符）
   - 标题通常以章节编号开头（如 "1."、"1.1"、"Chapter 1" 等）

#### 3.3 确定标题层级

根据字体大小和样式确定标题层级：

- **一级标题（H1）**：字体最大，通常对应章节标题（如 "第1章"、"1."）
- **二级标题（H2）**：字体次大，通常对应节标题（如 "1.1"）
- **三级标题（H3）**：字体略大，通常对应子节标题（如 "1.1.1"）
- 以此类推...

**层级判定算法：**

```
1. 收集所有候选标题的字体大小
2. 按字体大小降序排序
3. 计算相邻大小之间的比例
4. 根据比例突变点划分层级
5. 同层级内按页码和y坐标排序
```

### 4. 生成目录

根据识别出的标题结构生成目录。

#### 4.1 构建层级树

将扁平化的标题列表构建为树形结构：

```python
class TocEntry:
    def __init__(self, level, title, page_num, y_pos):
        self.level = level      # 目录层级（1-based）
        self.title = title      # 标题文本
        self.page_num = page_num  # 页码
        self.y_pos = y_pos      # 在页面中的y坐标
        self.children = []      # 子标题列表
```

#### 4.2 输出格式

支持以下输出格式：

**格式1：Markdown目录（推荐）**
```markdown
# 📑 PDF目录

## 第1章 引言 ........................ 1
### 1.1 研究背景 ...................... 3
### 1.2 相关工作 ...................... 5
## 第2章 方法 ........................ 10
### 2.1 数据预处理 .................... 12
### 2.2 模型架构 ...................... 15
```

**格式2：纯文本目录**
```
第1章 引言 ........................ 1
  1.1 研究背景 ...................... 3
  1.2 相关工作 ...................... 5
第2章 方法 ........................ 10
  2.1 数据预处理 .................... 12
  2.2 模型架构 ...................... 15
```

**格式3：JSON格式（便于程序处理）**
```json
[
  {"level": 1, "title": "第1章 引言", "page": 1, "children": [
    {"level": 2, "title": "1.1 研究背景", "page": 3},
    {"level": 2, "title": "1.2 相关工作", "page": 5}
  ]},
  {"level": 1, "title": "第2章 方法", "page": 10}
]
```

在 `skills/pdf-toc-generator/references/` 目录下生成参考Python脚本，包含核心实现代码。

参考文件路径：`skills/pdf-toc-generator/references/pdf_toc_generator.py`

### 6. 将结果写入输出文件

文件路径为 `automate_output/`

- 目录输出文件：`automate_output/[pdf文件名]_toc.md`（Markdown格式）
- 同时可输出JSON格式：`automate_output/[pdf文件名]_toc.json`

文件内容结构示例：

```markdown
# 📑 [PDF文件名] - 自动生成目录

> 生成时间：YYYY-MM-DD HH:MM:SS
> 总页数：N页 | 目录条目数：M条

---

## 目录

[生成的目录内容]

---

## 生成说明

- **识别方法**：基于字体大小、样式和位置特征自动识别标题
- **检测层级**：共检测到 X 级标题层次
- **置信度**：高/中/低（根据标题特征的明显程度评估）
```

### 7. 输出结果

向用户展示：
- PDF文件的基本信息（文件名、总页数）
- 检测到的标题层级数量
- 生成的目录预览（前几级标题）
- 保存的文件路径
- 目录质量评估（置信度）

### 8. 迭代优化

根据用户反馈进行调整：

1. **调整层级检测阈值**：如果层级划分不准确，调整字体大小比例阈值
2. **排除误识别**：过滤掉被误识别为标题的页眉、页脚、页码等
3. **手动指定层级**：允许用户指定某些文本应归属的层级
4. **补充遗漏标题**：手动添加未被识别的标题

## 注意事项

1. **依赖库**：需要安装 `pymupdf` 库（`pip install pymupdf`）
2. **PDF特性影响**：
   - 扫描版PDF（图片格式）无法直接提取文本，需要OCR预处理
   - 部分PDF的字体信息可能不完整，影响识别精度
   - 多列布局的PDF可能需要特殊处理
3. **标题识别局限性**：
   - 如果PDF中标题与正文字体大小差异不明显，识别效果可能不佳
   - 某些学术论文的标题格式不统一，可能需要人工校正
4. **页码准确性**：目录中的页码基于PDF内部的页面编号（从1开始），可能与PDF实际显示的页码不一致
5. **性能考虑**：处理大文件（数百页）时可能需要较长时间
